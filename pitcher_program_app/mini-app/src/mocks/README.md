# Backend-less dev mode (MSW)

Run the mini-app with **no backend** — every `/api/*` call is served from
fictional fixtures. Use this to iterate on UI, layout, and (soon) brand/colors
without touching the real backend, Supabase, or any athlete data.

```bash
npm run dev:mock
```

That sets two env vars and starts Vite:

- `VITE_USE_MOCKS=true` — `main.jsx` starts the MSW service worker before first
  render, so all `/api/*` requests are intercepted.
- `VITE_TEST_PITCHER_ID=mock_pitcher` — uses the existing Tier-3 auth fallback
  in `hooks/useTelegram.js`, so the app resolves a pitcher without Telegram.

You'll see `MSW mock backend active` in the browser console.

## Files

| File | Role |
|------|------|
| `fixtures.js` | Fictional response data ("Sample Pitcher" — no real roster) |
| `handlers.js` | MSW route handlers (wildcard origin) + a catch-all so any un-fixtured `/api/*` call returns a safe default instead of crashing a screen |
| `browser.js` | `setupWorker(...)` — loaded only when mocks are on |
| `__tests__/handlers.test.js` | Self-test: hits the handlers via `msw/node`, asserts shapes |

## Notes

- **Production is unaffected.** The worker is loaded via a dynamic `import()`
  guarded by `VITE_USE_MOCKS`, so MSW is never in the prod bundle and the real
  backend is used as normal when the flag is off.
- **Adding an endpoint:** add a fixture in `fixtures.js`, then a handler in
  `handlers.js` *above* the catch-all. Until you do, the catch-all returns `{}`
  (GET) / `{ ok: true }` (writes) so the screen still renders.
- `public/mockServiceWorker.js` is generated by `npx msw init public/ --save`;
  commit it so the worker is available without a postinstall step.
